STEPS TO CALL JIRA ASSETS WITH LOGIC APP
==========================================

PREREQUISITES
-------------
- Jira Cloud instance with Assets enabled
- Azure Logic App (Consumption or Standard)
- Jira API token (generated from your Atlassian account)


STEP 1: GENERATE A JIRA API TOKEN
----------------------------------
1. Go to https://id.atlassian.com > Security > Create and manage API tokens
2. Click "Create API token", give it a label, and copy the token


STEP 2: CREATE THE BASIC AUTH CREDENTIAL
-----------------------------------------
Jira Assets API uses Basic Auth: email:api_token encoded in Base64.

  Base64("your-email@domain.com:your-api-token")

Generate this in PowerShell:
  [Convert]::ToBase64String([Text.Encoding]::ASCII.GetBytes("email@domain.com:api-token"))

Store this in an Azure Key Vault secret or Logic App parameter.


STEP 3: ADD AN HTTP ACTION IN LOGIC APP
----------------------------------------
In your Logic App designer, add an HTTP action with:

  Method  : GET (or POST)
  URI     : https://api.atlassian.com/jsm/assets/workspace/{workspaceId}/v1/object/aql
  Headers :
    {
      "Authorization": "Basic <your-base64-encoded-credentials>",
      "Accept": "application/json",
      "Content-Type": "application/json"
    }


STEP 4: FIND YOUR WORKSPACE ID
--------------------------------
Call this endpoint first to get your workspaceId:

  GET https://api.atlassian.com/jsm/assets/workspace

The response returns a list of workspaces — grab the workspaceId value.


STEP 5: COMMON ASSETS API CALLS
---------------------------------
Search objects with AQL (Assets Query Language):
  POST https://api.atlassian.com/jsm/assets/workspace/{workspaceId}/v1/object/aql
  Body:
    {
      "qlQuery": "objectType = \"Server\" AND status = \"Active\""
    }

Get a specific object:
  GET https://api.atlassian.com/jsm/assets/workspace/{workspaceId}/v1/object/{objectId}

Get object types:
  GET https://api.atlassian.com/jsm/assets/workspace/{workspaceId}/v1/objecttype/all


STEP 6: PARSE THE RESPONSE
----------------------------
After the HTTP action, add a Parse JSON action:
  - Content : @body('HTTP')
  - Schema  : Generate from a sample response payload

Then use the parsed fields in subsequent Logic App steps.


STEP 7: HANDLE PAGINATION
---------------------------
Assets API returns paginated results. Add an Until loop:

  Until: @equals(body('HTTP')?['isLast'], true)

Inside the loop, increment the startAt query parameter:
  ?startAt=@{variables('offset')}&maxResults=50


TIPS
----
- Rate limits   : Atlassian enforces rate limits — add a Delay action if calling in bulk
- Error handling: Wrap the HTTP action in a Scope with Configure run after set to handle failures
- Managed Identity: If using Azure Key Vault for the token, grant the Logic App's managed
                    identity access to the secret
